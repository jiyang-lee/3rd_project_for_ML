# 모델 성능 검토 보고서 (pipeline · manufacturer 1)

## Overview

`pipeline/` 전처리 산출물로 고장 조기탐지 모델을 학습·검증하고, **성능을 올리는 여러 시도를
nested CV로 엄격히 검증**한 결과를 정리한다. 결론적으로 **MVP 0.25 → DHW+NaiveBayes 0.50**
(검증된 탐지율, 오경보 0.171)으로 2배 향상했고, 그 이상은 가용 데이터로 불가능함을 확인했다.

- 평가: `GroupKFold(설비단위)` OOF + 이벤트단위 K연속 규칙 (누수 방지)
- 검증: **Nested CV** — 모델·운영점을 train에서만 탐색 → 안 본 test에 적용
- 지표: 탐지율(고장 잡는 %↑) / 오경보율(정상 오판 %↓)
- 대상: 진짜 고장 28건 / 정상 35건

## 핵심 결과 (검증됨)

| 단계 | 데이터 | 탐지율 | 오경보율 | 잡는 고장 |
|---|---|---:|---:|---:|
| MVP | 공통센서 9 (55피처) | 0.25 | 0.20 | 7/28 |
| **+ DHW** | + 온수센서 (85피처) | **0.50** | **0.171** | **14/28** |

> best = **DHW + NaiveBayes**. NaiveBayes 단독 nested CV = 0.500 @ 0.171 (검증됨).

## 성능 향상 시도 전체 (in-sample vs 검증)

| 시도 | in-sample | **nested CV (검증)** | 판정 |
|---|---:|---:|---|
| ① 16모델 비교 (MVP) | — | best 0.214 | 상한 확인 |
| ② **DHW 추가** | 0.464 | **0.464~0.500** | ✅ 진짜 (2배) |
| ③ 추가 센서(rich) | — | 0.393 (하락) | ❌ 중복/희소 |
| ④ 모델별 최적화 | — | NaiveBayes 0.500 | ✅ 최고 |
| ⑤ OR 앙상블 | — | 0.500 (동일, FA↑) | ❌ 미탐 공유 |

핵심 교훈:
- **진짜 새 신호(DHW=온수 고장)는 검증 통과** → 일반화.
- **모델 트릭(앙상블)·중복 피처는 검증서 무효** → 과적합 or 이득 없음.
- 놓치는 14건은 **어느 모델도 못 잡음** = 센서에 신호가 없는 구조적 미탐.

## ★ 모델별 장단점 (DHW 데이터 검증 성적표)

nested CV로 각 모델을 개별 검증한 결과. **모델마다 탐지율-오경보 트레이드오프의 다른 지점**을 잡는다.

| 모델 | 탐지율 | 오경보 | 장점 | 단점 |
|---|---:|---:|---|---|
| **NaiveBayes** | **0.500** | 0.171 | 탐지율 최고. 온수/난방이 물리적으로 독립 → 독립가정 적중. 소표본 과적합 없음. 빠름 | 확률 보정 약함. 강상관 피처엔 취약(여기선 오히려 유리) |
| **ExtraTrees** | 0.357 | 0.229 | 균형형. 트리 앙상블로 견고, 결측·스케일 둔감 | 탐지·오경보 모두 중간. 해석 어려움 |
| **XGBoost** | 0.214 | **0.086** | **오경보 최저**(정상 35건 중 3건만). 보수적·정밀 | 탐지율 낮음. 소표본서 과적합 경향, 튜닝 필요 |
| **LightGBM** | 0.214 | 0.114 | 빠름·강력, 오경보 낮음 | 탐지율 낮음. 소표본 부적합 |
| **LogReg** | 0.214 | 0.143 | 해석 최고(계수), 안정적 확률, 과적합 적음 | 비선형·독립신호 활용 약함 |
| **SVM(RBF)** | 0.179 | 0.171 | 이론상 소표본 적합 | 확률보정 약함, 느림, 여기선 성능 하위 |

### 운영 목적별 추천
- **많이 잡고 싶다** → **NaiveBayes** (탐지 0.50, 오경보 0.17)
- **헛알람이 치명적이다** → **XGBoost** (오경보 0.086, 탐지 0.21)
- **균형** → **ExtraTrees** (0.357 / 0.229)
- **설명이 중요하다** → **LogReg** (계수로 근거 제시)

### 왜 NaiveBayes가 이겼나 (핵심 해석)
1. **피처 독립성**: 온수(DHW)와 난방(hc1)은 물리적으로 분리된 계통 → NaiveBayes의
   "피처 독립" 가정이 이 데이터에선 오히려 잘 맞음.
2. **소표본 강건성**: 독립 고장 28건뿐 → 파라미터가 적은 단순 모델이 과적합 없이 유리.
   부스팅·신경망은 여기서 과적합.
3. **결측 관용**: DHW 없는 설비(10/35)의 NaN을 imputation 후에도 안정적으로 처리.

## Verification

| 검증 | 결과 |
|---|---|
| MVP 16모델 (GroupKFold OOF) | best 탐지율 0.214 @ FA0.20 |
| DHW pooled nested CV | 0.464 @ 0.20 |
| DHW 모델별 nested CV | NaiveBayes 0.500 / ExtraTrees 0.357 / 부스팅 0.214 |
| 추가센서 rich nested CV | 0.393 (DHW 대비 하락) |
| OR앙상블 nested CV | 0.500 @ 0.229 (단독 못 넘음) |

## Limits and Caveats

- **표본 28건**: 탐지율 1건 = 0.036 → ±1~2건 노이즈. 방향·크기는 nested CV로 신뢰.
- **구조적 미탐 14건**: 제어 파라미터 오설정·"other" 등은 센서 파형에 안 드러남 → ML 원천 불가.
- **라벨 노이즈**: pre_fault가 `Possible anomaly start`(추정)·고정 horizon에 의존.
- manufacturer 2 미사용(범위 밖).

## 결론

- **검증된 최고 성능 = DHW + NaiveBayes, 탐지율 0.50 @ 오경보 0.17** (고장 14/28, 헛알람 6/35).
- MVP(0.25) 대비 2배. **DHW(진짜 새 신호)가 유일한 실효 지렛대**였다.
- 그 이상은 모델·조합·피처로 불가능(검증 완료). 남은 14건은 **센서 밖 데이터**(설정 변경 로그 등)가 있어야 잡힌다.

## Next Steps
